# Session Handoff Schema
Template for the Layer 3 session note written at the end of every dispatcher session. The next session reads this before anything else. Write this at close, not during.

## DECIDED
Locked decisions made this session. Do not re-litigate these.
Format: [decision] — [why it was locked]

## COMPLETED
What shipped this session. Include commit reference if applicable.
Format: [what] — [verified working / deployed / pending verification]

## OUTSTANDING
Mid-flight items, blocked items, follow-up actions needed before close.
Format: [item] — [what's needed to close it]

## NEXT SESSION BRIEF
Written for the next model or the next you. Standing-start instructions — not a to-do list.
- What to load first
- What the first task is
- Any gotchas before touching anything
- Estimated worker assignment for first task
